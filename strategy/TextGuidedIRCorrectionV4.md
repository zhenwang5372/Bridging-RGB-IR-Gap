# TextGuidedIRCorrectionV4 文本引导的红外特征纠错模块

## 1. 模块概述

`TextGuidedIRCorrectionV4` 是一个利用**文本语义信息**来指导**红外特征纠错**的模块。核心思想是：通过比较 RGB 和 IR 在文本语义激活上的差异，识别出 IR 特征中的"错误"或"偏差"区域，并对其进行自适应纠正。

### 设计动机

红外图像与 RGB 图像在同一场景下应该具有相同的语义内容，但由于成像原理不同，两者的特征表示可能存在差异。通过文本描述（如类别名称）作为统一的语义参考，可以：
1. 找出 IR 特征与 RGB 特征在语义响应上的不一致区域
2. 对这些区域进行自适应修正

---

## 2. 核心算法

### 2.1 算法步骤

1. **语义激活 (Semantic Activation)**
   - 使用文本特征作为 Query
   - 分别与 RGB 和 IR 特征计算 Attention Map

2. **一致性度量 (Consistency Measurement)**
   - 计算 RGB 和 IR 的 Attention Map 之间的一致性
   - 高一致性表示该区域的 IR 特征是可靠的
   - 低一致性表示该区域可能存在错误

3. **加权差异图 (Weighted Difference Map)**
   - 根据一致性分数，计算空间维度的错误分布图

4. **特征纠错 (Feature Correction)**
   - 使用可学习的 Alpha 参数控制纠错强度
   - V4版本：Alpha 初始值为 **-0.3**（负值意味着增强而非抑制）

---

## 3. 详细处理流程

```
输入:
  - rgb_feats: RGB特征 (P3, P4, P5)
  - ir_feats: IR特征 (P3, P4, P5)  
  - txt_feats: 文本特征 [B, num_cls, text_dim]

处理（对每个尺度独立进行）:

Step 1: 语义激活计算
┌────────────────────────────────────────────────────┐
│                                                    │
│  Text Features [B, N, D]                           │
│        │                                           │
│        ▼ Linear(D → d_k)                           │
│    Q [B, N, d_k]                                   │
│                                                    │
│  RGB Features [B, C, H, W]                         │
│        │                                           │
│        ▼ Conv1x1(C → d_k)                          │
│    K_rgb [B, d_k, H*W]                             │
│                                                    │
│  IR Features [B, C_ir, H, W]                       │
│        │                                           │
│        ▼ Conv1x1(C_ir → d_k)                       │
│    K_ir [B, d_k, H*W]                              │
│                                                    │
│  Attention:                                         │
│    A_rgb = Softmax(Q @ K_rgb / √d_k)  [B, N, H*W]  │
│    A_ir = Softmax(Q @ K_ir / √d_k)    [B, N, H*W]  │
│                                                    │
└────────────────────────────────────────────────────┘

Step 2: 一致性度量
┌────────────────────────────────────────────────────┐
│                                                    │
│  A_rgb_norm = L2_normalize(A_rgb, dim=-1)          │
│  A_ir_norm = L2_normalize(A_ir, dim=-1)            │
│                                                    │
│  G = sum(A_rgb_norm * A_ir_norm, dim=-1)           │
│  G ∈ [0, 1]  # 余弦相似度，越高越一致              │
│  G: [B, N]                                         │
│                                                    │
└────────────────────────────────────────────────────┘

Step 3: 加权差异图
┌────────────────────────────────────────────────────┐
│                                                    │
│  D_spatial = |A_rgb - A_ir|      [B, N, H*W]       │
│                                                    │
│  disagreement = 1 - G            [B, N]            │
│  # 不一致程度，越高说明该类别的IR表示越有问题       │
│                                                    │
│  M_err = einsum('bn,bnh->bh', disagreement, D)     │
│  # 加权求和，得到空间维度的错误图                  │
│  M_err: [B, H*W]                                   │
│                                                    │
│  M_err = normalize(M_err)        [B, 1, H, W]      │
│  # 归一化到 [0, 1]                                 │
│                                                    │
└────────────────────────────────────────────────────┘

Step 4: 空间门控与特征纠错
┌────────────────────────────────────────────────────┐
│                                                    │
│  F_extracted = X_ir * M_err_spatial                │
│  # 提取错误区域的特征                              │
│                                                    │
│  Error_map = ErrorEstimator(F_extracted)           │
│  # ErrorEstimator: Conv1x1 → BN → ReLU →           │
│  #                 Conv3x3 → BN → ReLU → Conv1x1   │
│                                                    │
│  ⭐ V4核心改动:                                    │
│  X_ir_corrected = X_ir - α × Error_map             │
│                                                    │
│  当 α = -0.3 时:                                   │
│  X_ir_corrected = X_ir + 0.3 × Error_map           │
│  # 负Alpha意味着增强而非抑制错误区域特征           │
│                                                    │
└────────────────────────────────────────────────────┘

输出:
  - rgb_feats: 原始RGB特征（不变）
  - ir_corrected_feats: 纠正后的IR特征
```

---

## 4. V4 版本的核心改进

### Alpha 初始值改为 -0.3

| 版本 | Alpha 初始值 | 效果 |
|------|-------------|------|
| V3 | +0.3 | 抑制错误区域 |
| V4 | **-0.3** | 增强错误区域 |

**理论依据**：
- 正值 Alpha：`X_ir - 0.3 × Error = 抑制`
- 负值 Alpha：`X_ir - (-0.3) × Error = X_ir + 0.3 × Error = 增强`

在某些场景下，IR 特征中被标记为"错误"的区域可能包含有用的补充信息，增强这些区域可能比抑制更有益。

## 5. 关键参数配置

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| `rgb_channels` | [128, 256, 512] | RGB特征通道数 |
| `ir_channels` | [64, 128, 256] | IR特征通道数 |
| `text_dim` | 512 | 文本特征维度 |
| `num_classes` | 4 | 类别数 |
| `correction_alpha` | -0.3 | 纠错强度初始值 (可学习) |
| `log_alpha` | True | 是否打印Alpha值 |
| `log_interval` | 50 | 打印间隔 |

---

## 6. Alpha 参数监控

模块内置了 Alpha 值的监控功能：

```python
# 训练时每50个iteration打印一次
[IR Correction V4] Iter [50] Alpha: P3=0.001234, P4=0.002345, P5=0.003456, Mean=0.002345

# 验证时每个epoch打印一次
[IR Correction V4] Val Alpha: P3=0.001234, P4=0.002345, P5=0.003456, Mean=0.002345
```

这帮助我们监控模型是否学习到了合适的纠错强度。

## 9. 设计优势

1. **文本语义指导**：利用文本作为统一的语义参考，避免直接比较不同模态的特征
2. **自适应纠错**：可学习的 Alpha 参数允许模型学习最优的纠错强度
3. **多尺度处理**：独立处理 P3、P4、P5 三个尺度
4. **内置监控**：训练过程中自动打印 Alpha 值，便于调试
5. **负值初始化**：V4 的负 Alpha 初始值提供了特征增强的可能性
